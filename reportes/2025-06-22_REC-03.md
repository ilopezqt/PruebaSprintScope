## Resumen

Spec técnica completa del sistema de recomendación, incorporando los hallazgos del EDA de Persona2.

## Arquitectura decidida

Sistema híbrido en dos capas:
1. **Capa colaborativa (ALS)**: para usuarios con ≥5 interacciones. Modelo entrenado semanalmente en batch.
2. **Capa content-based**: para usuarios con <5 interacciones (cold start). Basada en similitud de productos por categoría, precio y descripción.

Combinación: weighted ensemble con α=0.7 colaborativo + 0.3 content-based, ajustable por configuración.

## API

```
POST /recommendations/{user_id}
Query params: n (default 10), strategy (auto|collaborative|content)
Response: { recommendations: [{product_id, score, strategy_used}], latency_ms }
```

## Lo que funcionó

Incorporar los hallazgos del EDA directamente en la spec evitó tener que revisarla después. La decisión de ir con un híbrido desde el principio está bien fundamentada.

## Decisiones pendientes

La estrategia de serving (real-time vs batch+caché) queda abierta hasta que Persona1 y Persona4 evalúen la latencia durante el desarrollo.
