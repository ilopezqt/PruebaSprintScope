## Resumen

Setup del repositorio y estructura base del proyecto. Implementación del esqueleto de la arquitectura híbrida definida en la spec.

## Lo implementado

- Estructura de carpetas: `src/models/`, `src/api/`, `src/data/`, `src/evaluation/`
- Clase base `BaseRecommender` con interfaz común para colaborativo y content-based
- `HybridRecommender` que orquesta ambos modelos con el peso α configurable
- FastAPI app skeleton con el endpoint `/recommendations/{user_id}` (stub que devuelve datos mock)
- Docker Compose con PostgreSQL para almacenar las recomendaciones precalculadas
- 12 tests unitarios de la interfaz base

## Lo que funcionó

La interfaz común `BaseRecommender` hace que sea muy fácil añadir nuevos modelos sin tocar el código de orquestación. La decisión de usar ABC de Python para enformar la interfaz fue buena.

## Problemas encontrados

Ninguno relevante. El setup fue directo.

## Decisiones de diseño

- α=0.7 como default pero configurable por variable de entorno `HYBRID_ALPHA`
- El endpoint devuelve siempre `strategy_used` para facilitar el debugging y la evaluación A/B posterior
